Submission format

What you submit

Submit a Transfer_policy class, initialized with the building's metadata and action spec. Actions are drawn from it via get_action.

The interface

Train a Transfer_policy

Your submission is a policy class initialized with building_meta_data and action_spec. At each step, the evaluation harness calls get_action(obs) to obtain the discrete on/off commands and continuous setpoints for the current building. Because the metadata carries the action dimensions, one class handles any building.

transfer_policy.py 
from typing import Dict, Any
import numpy as np


class Transfer_policy:
    def __init__(
        self,
        building_meta_data,
        action_spec,
    ):
        self.metadata = building_meta_data
        self.action_spec = action_spec
        self.n_discrete = building_meta_data["n_discrete_action"]
        self.n_continuous = building_meta_data["n_continuous_action"]

    @property
    def name(self) -> str:
        return "transfer"

    def get_action(self, obs: np.ndarray) -> Dict[str, Any]:
        return {
            "continuous_action": np.zeros(self.n_continuous, dtype=np.float32),
            "discrete_action": np.zeros(self.n_discrete, dtype=np.int32),
        }

Reference policy. A minimal Transfer_policy that reads its action dimensions from the building metadata and returns a zero action. Replace get_action with your learned policy.

Building metadata

How one policy adapts to any building

The metadata is what lets a single policy condition its control on the specific building it operates, rather than rely on a fixed input layout.

Action structure

The number of discrete and continuous actions, and the indices of the temperature, pressure, and water setpoints within the continuous action vector.

Observation structure

The observation field names and the indices of the zone temperature sensors.

Building layout

The floor plan, the zone map, the assignment of zones to AHUs, and the number of floors.

Schedule & comfort targets

Morning and evening start hours, the comfort window during work hours, the eco window for unoccupied periods, the time zone, and the holiday calendar.

Normalization

Observation and action normalization metadata that place raw sensor readings and setpoints on a common scale.

Why it matters

Because the floor plan, zone map, and zone-to-AHU assignments are delivered for every building, a policy can parse each building's structure and adapt, which is exactly the capability the competition rewards.
